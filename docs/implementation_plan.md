# План Реализации Автоматизации ANSYS Mechanical

## Обзор
Данный документ описывает детальный план реализации скриптов для автоматизации расчетов в ANSYS Mechanical.

## Статус проекта
План реализован. Основные компоненты (менеджеры) настроены и функционируют. Выполнены исправления:
1. Автоматическое переименование контактов (`RenameBasedOnDefinition`).
2. Исправлена логика настройки сетки для тел с суффиксами (regex match).
3. Исправлена передача объектов `Quantity` и Enums через `ProjectContext` для избежания конфликтов `TypeError`.
4. Реализована настройка `SurfaceMeshMethod` и `FreeMeshType` через строковые значения в `mesh_config.json`.
5. Реализована настройка `Remote Displacement` (вращения и перемещения) с корректной обработкой `quantity` и шагов анализа.
6. Настроено применение нагрузок (Force) для `gu_force`.

## Именование параметров проекта
- **ID проекта**: Парсится из названия проекта ANSYS (`Project_XXX`) для загрузки соответствующего файла конфигурации `structure_type_XXX.json`.
- **Параметры исполнения и диаметра**: Извлекаются из названия модели (`self.model.Name`) по шаблону `NN-DDD` (где NN - номер исполнения, DDD - диаметр трубопровода). Эти данные используются менеджером анализа для выбора соответствующих нагрузок из конфигурации.
