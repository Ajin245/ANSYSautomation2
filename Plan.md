## Итоговый Алгоритм Рабочего Процесса Автоматизации ANSYS

Этот алгоритм описывает сквозной процесс автоматизации, включающий подготовку геометрии в SpaceClaim, настройку анализа в Mechanical и постобработку результатов.

### 0. Необходимые исходные данные от пользователя
Для успешного выполнения алгоритма от пользователя потребуются следующие данные:

*   **0.1. Идентификатор Конструкции (`XXX`):**
    *   **Описание:** Уникальный идентификатор, который будет использоваться для выбора соответствующего файла `structure_type_XXX.json`.
    *   **Пример:** `151`, `custom_001`.

*   **0.2. Файл Конфигурации Конструкции (`structure_type_XXX.json`):**
    *   **Описание:** JSON-файл, содержащий все специфичные для данной конструкции настройки.
    *   **Требования к содержимому (минимум):**
        *   **Тип конструкции:** (строка)
        *   **Болтовые нагрузки:**
            *   Список болтов с их диаметрами и **моментами затяжки** (из которых будет вычисляться преднатяг).
            *   Правила именования Named Selections для болтов (если отличаются от общих).
        *   **Граничные условия:**
            *   Типы граничных условий (Fixed Support, Displacement и т.д.) и соответствующие Named Selections.
        *   **Нагрузки:**
            *   Применяемые силы, моменты, давления, с указанием Named Selections и значений.
            *   Многошаговые сценарии нагрузок (коэффициенты).
        *   **Named Selections для SpaceClaim:** Правила или паттерны для автоматического создания Named Selections в SpaceClaim.

*   **0.3. Дополнительные данные (опционально):**
    *   **Описание:** Любые специфические параметры, которые могут потребоваться, но не включены в `structure_type_XXX.json` (например, нестандартные свойства материалов, если они не заданы в исходной модели).

### 1. Централизованная Конфигурация Проекта

*   **1.1. Конфигурация Конструкции (`structure_type_XXX.json`)**
    *   **Назначение:** Единый источник истины для конкретного типа рассчитываемой конструкции, иногда частично переопределяющий общие настройки.
    *   **Расположение:** `config/structure_type_XXX.json` (где `XXX` - идентификатор типа конструкции).
    *   **Содержимое:**
        *   Тип конструкции и её характеристики.
        *   Параметры, **переопределяющие** `project_settings.json` (например, специфичные для конструкции граничные условия или нагрузки).
        *   **Полные данные по нагрузкам и моментам затяжки резьбовых соединений** (заменяет `bolt_database.json` и `load_database.json`).
    *   **Доступ:** Все скрипты (SpaceClaim и Mechanical) будут обращаться к этому файлу.

*   **1.2. Общие Конфигурации Mechanical (`config/mechanical_configs/*.json`)**
    *   **Назначение:** Общие настройки для Mechanical, не зависящие от конкретной конструкции.
    *   **Расположение:** `config/mechanical_configs/` (будут содержать `mesh_settings.json`, `contact_settings.json`, `analysis_scenarios.json`, `project_settings.json` и т.д.).

*   **1.3. Принципы Доступа к Конфигурации:**
    *   Все скрипты должны динамически определять пути к конфигурационным файлам (например, относительно текущего местоположения скрипта или через переменные окружения). **Жесткие ссылки на папку `Examples/` недопустимы.**

### 2. Этап Подготовки Геометрии в ANSYS SpaceClaim (SC)
*   **Цель:** Подготовка 3D-модели для анализа в Mechanical с использованием новой кодовой базы SpaceClaim.
*   **Действия пользователя:**
    *   Открытие исходной 3D-модели в SpaceClaim.
    *   Запуск **новых скриптов SpaceClaim** (расположенных, например, в `src/SpaceClaim/`) **последовательно**. Скрипты из `Examples/SpaceClaim/` служат примерами для разработки.
*   **Автоматизированные действия (новые скрипты SC):**
    *   **Чтение конфигурации:** Скрипты SC динамически обращаются к `config/structure_type_XXX.json` для получения параметров.
    *   **Очистка модели:** Удаление "мусора", упрощение геометрии согласно правилам.
    *   **Дополнительные построения:** Создание необходимых геометрических элементов.
    *   **Переименование тел:** Приведение имен тел в соответствие с соглашениями для работы скриптов Mechanical.
    *   **Назначение Named Selections:** Автоматическое создание всех требуемых Named Selections на основе правил из `config/structure_type_XXX.json` и логики новых скриптов SpaceClaim.
*   **Результат:** Чистая, подготовленная модель с корректными Named Selections, готовая к импорту в Mechanical.

### 3. Этап Настройки Анализа в ANSYS Mechanical
*   **Цель:** Полная настройка расчетной модели и граничных условий с использованием новой кодовой базы Mechanical.
*   **Действия пользователя:**
    *   Импорт подготовленной геометрии из SpaceClaim в Mechanical.
    *   Запуск **нового основного скрипта автоматизации** (например, `src/Mechanical/ansys_automation.py`). Скрипт `ansys_automation_v01.5.py` из `Examples/Mechanical/` служит примером для разработки.
*   **Автоматизированные действия (новый основной скрипт):**
    *   **Загрузка конфигурации:**
        *   Загрузка общих конфигураций из `config/mechanical_configs/*.json` (`mesh_settings.json`, `contact_settings.json`, `analysis_scenarios.json`, `project_settings.json`).
        *   Загрузка и **обработка `config/structure_type_XXX.json`**, где его параметры **переопределяют** соответствующие из `project_settings.json` и содержат все данные по нагрузкам и болтам.
    *   **Назначение Контактов:** `ContactManager` настраивает контактные пары на основе `contact_settings.json` и переопределений из `structure_type_XXX.json`.
    *   **Настройка Сетки КЭ:** `MeshManager` применяет настройки сетки на основе `mesh_settings.json` и Named Selections.
    *   **Назначение Граничных Условий:** `AnalysisManager` применяет граничные условия, используя `project_settings.json` и `structure_type_XXX.json`.
    *   **Назначение Нагрузок:** `AnalysisManager` и `BoltManager` применяют нагрузки, используя `structure_type_XXX.json` (расчет преднатяга болтов) и `analysis_scenarios.json`.
    *   **PlotResults (предварительная оценка):** `ResultsManager` создает объекты результатов для визуальной оценки корректности расчета:
        *   Общие перемещения.
        *   Напряжения в швах.
        *   Напряжения в болтах.
        *   Напряжения в конструкции в целом (для основных тел, исключая трубу, шайбы, гайки, швы).
*   **Результат:** Полностью настроенная модель Mechanical с граничными условиями, нагрузками, сеткой и предварительными результатами для проверки.

### 4. Этап Расчета
*   **Действия пользователя:**
    *   Запуск расчета в ANSYS Mechanical.
*   **Результат:** Решенная модель.

### 5. Этап Постобработки Результатов
*   **Цель:** Автоматизированное извлечение и выгрузка ключевых результатов с использованием новой кодовой базы постобработки.
*   **Действия пользователя:**
    *   Запуск **нового макроса постобработки** (например, из `src/Mechanical/postprocessing.py`). Скрипты `ResultsScript.py` и `FandM_ReactionsScript.py` из `Examples/Mechanical/` служат примерами для разработки.
*   **Автоматизированные действия (новый макрос):**
    *   **Извлечение результатов:** Создание и вычисление необходимых результатов (например, реакции опор, максимальные напряжения в ключевых зонах), основываясь на логике из `config/structure_type_XXX.json` и `config/mechanical_configs/project_settings.json`.
    *   **Выгрузка данных:** Экспорт данных в удобный формат (например, CSV) для дальнейшего анализа.
*   **Результат:** Выгруженные данные результатов для отчета и анализа.
